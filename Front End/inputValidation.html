HTML input validation

browser's built-in ability to check user inputs against rules of the product that we define using the html attributes, ZERO JS REQUIRED!

This is the fastest, smoothest and simplest form of validation i.e. the first layer of security before the data reaches our server

Why is it important

  • It prevents empty inputs and malicious inputs from being sent to the server
  • Works in all browsers
  • Improves UX feedback without page reloads

Validation Attributes

Attributes Purpose Works on
required field must not be empty text, email, number, passowrd, etc etc
type Enforces the data format input tag
minLength Minimum number of character in input text, password, email
maxLength Maximum characters text, pass, email
min Minimum numeric value number, range,date, time
max Maximum numeric value number, range,date, time
pattern REGEX text, pass, email, etc
title tooltip: the hint shown when you hover on the input
multiple for multiple values (comma seperated) email, file
step least applicable increment value number, range, date
