REST API -> Representation State Transfer

Its a set of conventions for designing web APIs that let clients communicate with a server through HTTP 

A REST api models data as resources.
- Users -> /users not /getUSers or /createUsers 
- Products -> /products not /findProducts or /buyProducts


REST simply states -> any request made by the client to the server MUST contain everything that is required by the server to process it, and in a very structured and clean way, this promotes scalability and security of the data and the server 


1. Statelessness each request must contain everything the server needs to process it. 
2. always use nounds in URL,  not verbs 
Correct.                 Avoid.
domain/users GET.        domain/getUsers GET
domain/users POST.       domain/createUser POSt
3. Use HTTP methods carefully
    GET -> read 
    POST -> create 
    PUT -> replace something
    PATCH -> update a part of something
    DELETE-> remove something
4. Use meaningful resource path names 
        GET /products
        GET /products/10
